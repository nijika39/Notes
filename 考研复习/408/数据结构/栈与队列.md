# 栈与队列

本篇笔记整理了栈和队列在 408 考研中的核心易错概念，以及高频考查的代码优化技巧。

---

## 1. 易错点

### 1.1 队头队尾指针问题

#### ① 链表头指针与队列首结点的区别
* **链式队列**中，头指针（通常对应 `front`）指向单链表的首个结点（如果是带头结点的链表，则指向头结点）。
* **重要考点**：用单链表实现队列时，**队头（front）必须设置在链表的头部**。因为出队操作在头部进行只需要 $O(1)$；如果在尾部出队，则需要遍历整条链表去寻找尾结点的前驱，时间复杂度会退化为 $O(n)$。

#### ② 无头结点的链式队列出队特殊情况
* **注意**：对于**不带头结点**的链式队列，在执行出队（Dequeue）时，若队列中**只剩最后一个结点**：
  在释放该结点空间后，必须手动修改尾指针，使：
  $$\text{rear} = \text{front} = \text{NULL}$$
  否则 `rear` 会变成野指针。

## 2. 循环队列的三种设计方案

> ⚠️ **前提条件**：以下三种方案均采用 **“牺牲一个存储单元”** 的方法来区分队空与队满。

### 方案 a)：`front` 指向队首，`rear` 指向队尾的后一个位置（最常用）
* **初始化**：
  $$\text{front} = \text{rear} = 0$$
* **判定队空**：
  $$\text{front} == \text{rear}$$
* **判定队满**：
  $$(\text{rear} + 1) \% \text{MaxSize} == \text{front}$$

### 方案 b)：`front` 指向队首的前一个位置，`rear` 指向队尾
* **初始化**：
  $$\text{front} = \text{rear} = 0$$
* **判定队空**：
  $$\text{front} == \text{rear}$$
* **判定队满**：
  $$(\text{rear} + 1) \% \text{MaxSize} == \text{front}$$

### 方案 c)：`front` 指向队首，`rear` 指向队尾
* **初始化**：
  $$\text{front} = 0, \quad \text{rear} = \text{MaxSize} - 1$$
* **判定队空**：
  $$(\text{rear} + 1) \% \text{MaxSize} == \text{front}$$
* **判定队满**：
  $$(\text{rear} + 2) \% \text{MaxSize} == \text{front}$$

---

### 💡 归纳总结

1. **公式完全一致性**：方案 a) 和 方案 b) 的初始化、判空、判满公式**完全一致**，它们唯一的区别在于入队（Enqueue）时 `rear` 的行为有所不同：
   * 方案 a) 先送值写入 `rear` 所指位置，再让 `rear` 后移。
   * 方案 b) 先让 `rear` 后移，再送值写入。
2. **数组与链表实现建议**：
   * 用**数组**实现循环队列时，方案 a) 和 方案 b) 均可使用。
   * 用**链表**实现循环队列时，**宜采用方案 b)**（即采用带头结点的链表，此时 `front` 刚好指向头结点，队首元素在头结点的下一个结点位置）。
