# 最小生成树的性质和算法

本章主要介绍最小生成树（MST）的性质和两种经典的最小生成树算法：Prim算法和Kruskal算法。

---

## 一、 最小生成树的性质

> [!warning] 核心性质（选择题高频考点）
> 1. **唯一性判断**：
>    * 若图 $G$ 中存在**权值相同**的边，其最小生成树**可能不唯一**。
>    * 当图 $G$ 中所有边的**权值互不相同**时，其最小生成树是**唯一**的。
>    * 当图 $G$ 中任意一个环所包含的边**权值互不相同**时，其最小生成树是**唯一**的。
>    * 即使最小生成树不唯一，所有最小生成树的**总权重（边权之和）也必然相同**。
> 2. **最短路径非等价性**：
>    * 最小生成树仅保证整棵树的**边权之和最小**，但**不能**保证树中任意两点之间的路径是图中的最短路径。
> 3. **顶点与边数关系**：
>    * 对于含有 $n$ 个顶点的连通图，其最小生成树必然含有且仅含有 $n-1$ 条边。

---

## 二、 最小生成树算法

这两个算法都基于**贪心策略（Greedy Strategy）**，且只适用于 **`[[带权无向图]]`**。

### 1. Prim 算法（普里姆算法）

#### 算法步骤：
1. 初始化：设置一个顶点集合 $U = \{v_0\}$，其中 $v_0$ 为起始源点；剩余顶点集为 $V - U$。
2. 贪心选择：在所有连接 $U$ 中顶点与 $V-U$ 中顶点的边中，挑选一条**权值最小**的边 $(u, v)$，将顶点 $v$ 加入集合 $U$，边 $(u, v)$ 加入生成树边集。
3. 更新状态：更新其余未加入顶点到当前集合 $U$ 的最小距离。
4. 重复上述步骤 2、3，直到 $U = V$（即所有顶点均已加入生成树）。

#### C 语言代码实现（基于 `[[邻接矩阵]]`）：
```c
#define INF 0x3f3f3f3f
#define MaxVertexNum 100

// 图的邻接矩阵定义
typedef struct {
    int Edge[MaxVertexNum][MaxVertexNum];
    int vexnum, arcnum;
} MGraph;

// Prim算法实现
void Prim(MGraph G, int v0) {
    int lowcost[MaxVertexNum];  // 保存各顶点到当前生成树顶点的最小边权
    int adjvex[MaxVertexNum];   // 记录该最小边在生成树中的双亲顶点
    
    // 1. 初始化数据
    for (int i = 0; i < G.vexnum; i++) {
        lowcost[i] = G.Edge[v0][i];
        adjvex[i] = v0;
    }
    lowcost[v0] = 0; // 标记顶点 v0 已加入生成树

    // 2. 循环 n-1 次，每次并入一个顶点
    for (int i = 1; i < G.vexnum; i++) {
        int min = INF;
        int k = -1;
        
        // 寻找当前未加入树的顶点中 lowcost 最小的顶点 k
        for (int j = 0; j < G.vexnum; j++) {
            if (lowcost[j] != 0 && lowcost[j] < min) {
                min = lowcost[j];
                k = j;
            }
        }
        
        if (k != -1) {
            // 打印/记录加入的边（实际考试中常考手工模拟）
            printf("边 (%d, %d) 加入最小生成树，权值：%d\n", adjvex[k], k, min);
            lowcost[k] = 0; // 标记顶点 k 已加入生成树
            
            // 3. 重新计算未加入树的顶点到当前生成树的最小距离
            for (int j = 0; j < G.vexnum; j++) {
                if (lowcost[j] != 0 && G.Edge[k][j] < lowcost[j]) {
                    lowcost[j] = G.Edge[k][j];
                    adjvex[j] = k;
                }
            }
        }
    }
}
```

### 2. Kruskal 算法（克鲁斯卡尔算法）

#### 算法步骤：
1. 初始化：将图 $G$ 中的所有边按权值**从小到大**进行排序。
2. 初始化并查集：使每个顶点自成一个独立的连通分量（等同于 $n$ 棵独立的树）。
3. 贪心选择：依次遍历排好序的边 $(u, v)$：
   * 利用 **`[[并查集]]`** 判断 $u$ 和 $v$ 是否属于同一个连通分量。
   * 若不属于（不构成回路），则将该边并入生成树，并合并 $u$ 和 $v$ 所在的连通分量。
   * 若属于（会构成回路），则舍弃该边。
4. 重复步骤 3，直到生成树中已有 $n-1$ 条边。

#### C 语言代码实现（结合并查集）：
```c
#include <stdio.h>

// 边的结构体定义
typedef struct {
    int u, v;       // 边的两个顶点
    int weight;     // 边的权值
} Edge;

// 并查集：查找根节点（路径压缩优化版）
int Find(int parent[], int i) {
    if (parent[i] < 0) 
        return i;
    return parent[i] = Find(parent, parent[i]); // 路径压缩
}

// 并查集：合并两个集合
void Union(int parent[], int x, int y) {
    int root1 = Find(parent, x);
    int root2 = Find(parent, y);
    if (root1 != root2) {
        // 简单合并，root2 作为 root1 的子树
        parent[root1] = root2; 
    }
}

// 假设 edges 已经按权值从小到大排好序
void Kruskal(int vexnum, Edge edges[], int edgeNum) {
    int parent[MaxVertexNum];
    
    // 初始化并查集：每个顶点代表一棵独立的树（值为-1表示根节点）
    for (int i = 0; i < vexnum; i++) {
        parent[i] = -1;
    }
    
    int edge_count = 0; // 记录已加入生成树的边数
    
    for (int i = 0; i < edgeNum; i++) {
        int u = edges[i].u;
        int v = edges[i].v;
        
        // 检查两个顶点是否属于同一集合（是否构成回路）
        if (Find(parent, u) != Find(parent, v)) {
            printf("边 (%d, %d) 加入最小生成树，权值：%d\n", u, v, edges[i].weight);
            Union(parent, u, v); // 合并集合
            edge_count++;
            
            // 选够 n-1 条边即可提前结束
            if (edge_count == vexnum - 1) {
                break;
            }
        }
    }
}
```

## 三、 Prim 算法和 Kruskal 算法对比

在 408 考研中，算法的选择和时空复杂度是选择题常客。

| 对比维度 | Prim 算法（普里姆） | Kruskal 算法（克鲁斯卡尔） |
| :--- | :--- | :--- |
| **算法思想** | **从顶点开始**拓扑式生长（点驱动型） | **从边开始**挑最小权边组合（边驱动型） |
| **时间复杂度（常规）**| $O(\|V\|^2)$ | $O(\|E\| \log \|E\|)$ |
| **时间复杂度（堆优化）**| $O(\|E\| \log \|V\|)$（使用 **`[[堆]]`** 优化后） | — |
| **空间复杂度** | $O(\|V\|)$ （辅助数组大小） | $O(\|E\|)$ （保存边和并查集空间） |
| **适用图类型** | **边稠密图**（顶点少、边多的图） | **边稀疏图**（顶点多、边少的图） |
| **检测回路方式** | 无需检测（只在安全顶点集合外挑选） | 必须使用 **`[[并查集]]`** 判断是否成环 |
| **适用图属性** | 带权无向图 | 带权无向图 |
