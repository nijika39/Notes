# 树的应用：哈夫曼树与并查集

哈夫曼树用于最优编码和数据压缩，并查集用于高效处理不相交集合的合并与查询问题。

## 1. 哈夫曼树 (Huffman Tree)

* **一句话定义**：在含有 $n$ 个带权叶子结点的二叉树中，其中带权路径长度（WPL）最小的二叉树，也称**最优二叉树**。

### 1.1 二叉哈夫曼树的性质
1. **带权与路径**：每个初始结点最终都会成为叶子结点。权值越小的结点，到根结点的路径长度越大。
2. **结点总数**：在构造过程中，共新建了 $n-1$ 个双分支结点，故整棵哈夫曼树的总结点数为：
   $$2n - 1$$
3. **结点度数**：哈夫曼树是正则二叉树，不存在度为 $1$ 的结点（每个新结点的度必然为 $2$）。
4. **WPL计算技巧**：哈夫曼树的带权路径长度（WPL）等于**所有非叶结点（新生成的双分支结点）的权值之和**。

### 1.2 多叉哈夫曼树（度为 $m$）
1. 只含有度为 $0$ 和度为 $m$ 的结点。
2. **虚结点的引入**：
   若初始结点数为 $n$，构建度为 $m$ 的哈夫曼树时，需要进行 $\lceil \frac{n-1}{m-1} \rceil$ 次合并。为了保证每次合并都是满 $m$ 叉，我们需要补充权值为 $0$ 的“虚结点”，补充数量为：
   $$(m - 1) - (n - 1) \% (m - 1)$$
   *(注意：若 $(n-1) \% (m-1) == 0$，则无需补充虚结点。)*

### 1.3 哈夫曼编码 (Huffman Coding)
* **前缀编码判定**：判定一组编码是否具有前缀特性（即任何一个编码都不是另一个编码的前缀），可以通过构建二叉树来完成。
* **判定算法思想**：
  从根节点出发，依次读入编码 $C$：
  * 若遇到非叶结点，说明有其他编码是当前编码 $C$ 的前缀，判定失败。
  * 若处理完 $C$ 的所有位时未创建新结点，说明 $C$ 是已有其他编码的前缀，判定失败。
  * 若在最后一位正常创建了新结点，则继续验证下一个编码。

## 2. 并查集 (Disjoint Set)

* **一句话定义**：一种用于管理和合并相互不重叠的集合，并支持快速查询所属集合的树形数据结构。

### 2.1 并查集核心性质
1. **存储结构**：采用树的**[[双亲表示法]]**进行存储，通常用数组 `S[]` 实现。
2. **根结点表示**：根结点的双亲域 `S[root]` 存储**负数**，其绝对值 $|S[root]|$ 代表该集合中元素的总数量。
3. **复杂度分析**：
   * 未优化时，`Find` 和 `Union` 的时间复杂度取决于树的深度 $O(d)$。
   * 使用 **“小树并大树”** 优化 `Union` 后，树的最大深度不超过：
     $$\lfloor \log_2 n \rfloor + 1$$
   * 在此基础上，引入 **[[路径压缩]]** 优化后，查找操作的平均深度不超过 $O(\alpha(n))$（$\alpha(n)$ 为阿克曼反函数，实际应用中 $\alpha(n) \le 4$）。

### 2.2 核心算法实现 

这里给出了408手考中最高频的两个并查集优化算法实现：

#### 1. 路径压缩查找 (迭代法)
两趟扫描法：第一趟找到根结点，第二趟将查找路径上的所有结点的双亲直接指向根结点。

```c
int Find(int S[], int x) {
    int root = x;
    // 第一趟：顺着双亲指针向上找到根结点
    while (S[root] >= 0) {
        root = S[root]; 
    }
    // 第二趟：路径压缩（将路径上的所有结点的双亲设为根结点）
    int curr = x;
    while (curr != root) {
        int parent = S[curr]; // 暂存当前结点的原本双亲
        S[curr] = root;       // 挂载到根节点上
        curr = parent;        // 继续处理原本的双亲结点
    }
    return root; // 返回根结点下标
}
```

#### 2. 路径压缩查找 (递归法)
利用递归在查找根结点的同时，将路径上的所有结点的父指针直接指向根结点。

```c
int Find(int S[], int x) {
    if (S[x] < 0) {
        return x; // 找到根节点
    } else {
        // 递归查找，并将沿途结点的父节点直接设为根节点（路径压缩）
        return S[x] = Find(S, S[x]); 
    }
}
```

#### 3."小树并大树"合并算法
根据根结点的负数值（绝对值代表结点总数）进行判定，始终让结点少的树作为子树并入结点多的树，防止树退化。
```c
void Union(int S[], int root1, int root2) {
    if (root1 == root2) return; // 属于同一集合，无需合并

    // 注意：S[root] 存储负数，绝对值表示元素个数
    // 负数越小，代表含有的结点越多（例如 S[root1]=-5, S[root2]=-3，说明root1树更大）
    if (S[root1] > S[root2]) { 
        // root2 树更大，将 root1（小树）并入 root2（大树）
        S[root2] += S[root1]; // 累加结点总数
        S[root1] = root2;     // 小树根指向大树根
    } else {
        // root1 树更大或相等，将 root2（小树）并入 root1（大树）
        S[root1] += S[root2];
        S[root2] = root1;
    }
}
```
