# KMP 算法

KMP 算法是一种改进的 **[[字符串模式匹配算法]]**。它的核心思想是：利用已经匹配过的部分匹配信息，在发生失配时避免主串指针回溯，而是通过移动子串（模式串）的指针继续匹配，从而将时间复杂度降低到 $O(m + n)$。通常与经典的 **[[BF 算法 (朴素模式匹配)]]** 作对比。

## 1. 名词解释

* **PM 数组 (Partial Match, 部分匹配值数组)**
  * **一句话定义**：记录模式串中每个字符及之前子串的**[[最长相等前后缀]]**长度。
  * **手写释义**：指子串中第 $j$ 个字符及之前的串中，最长相等前后缀的长度。
* **next 数组**
  * **一句话定义**：指示当模式串第 $j$ 个字符与主串失配时，主串不回溯，模式串应回退到的匹配位置。
  * **手写释义**：指第 $j$ 个位置的字符失配时，主串中相应字符重新与子串中第 $\text{next}[j]$ 个字符进行匹配。
* **nextval 数组**
  * **一句话定义**：对 $\text{next}$ 数组的进一步优化，解决了当“失配字符”与“回溯后的字符”相同时会导致二次必然失配的冗余问题。
  * **手写释义**：修正后的 $\text{next}[j]$ 数组。

---

## 2. 手算方法与公式

> ⚠️ **注意**：以下数组下标均从 **$1$** 开始。

### 核心公式

1. **求 next 数组**：
   $$\text{next}[j] = \text{PM}[j-1] + 1$$
   *(特别地，规定 $\text{next}[1] = 0$)*
2. **求 nextval 数组**：
   * 若当前字符 $T[j]$ 与回溯字符 $T[\text{next}[j]]$ **不同**，则照抄原值：
     $$\text{nextval}[j] = \text{next}[j]$$
   * 若当前字符 $T[j]$ 与回溯字符 $T[\text{next}[j]]$ **相同**，则需要继续追溯：
     $$\text{nextval}[j] = \text{nextval}[\text{next}[j]]$$
3. **计算滑动长度**：
   $$\text{滑动长度} = j - \text{nextval}[j] \quad (\text{或者 } j - \text{next}[j])$$

---

### 手算示例

设模式串 $T = \text{"ababaab"}$，各数组推导如下表所示：

| 字符序号 $j$ | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 备注说明 |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :--- |
| **模式串 $T[j]$** | `a` | `b` | `a` | `b` | `a` | `a` | `a` | |
| **PM 数组** | 0 | 0 | 1 | 2 | 3 | 1 | 1 | 最长相等前后缀长度 |
| **next 数组** | 0 | 1 | 1 | 2 | 3 | 4 | 2 | $\text{next}[j] = \text{PM}[j-1] + 1$ |
| **相应字符 $T[\text{next}[j]]$** | - | `a` | `a` | `b` | `a` | `b` | `b` | 用于计算 nextval 的辅助行 |
| **nextval 数组** | 0 | 1 | 0 | 1 | 0 | 4 | 2 | 字符不同抄 next；字符相同抄对应 nextval |
| **滑动长度** | 1 | 1 | 3 | 3 | 5 | 2 | 5 | $j - \text{nextval}[j]$ |

#### 针对 $\text{nextval}$ 的手算推导解析：
* **$j=1$**：首字符直接规定为 $0$。
* **$j=2$**：$T[2] = \text{'b'}$ 与 $T[\text{next}[2]] = T[1] = \text{'a'}$ **不同**，照抄 $\text{next}[2]$ 得到 $\mathbf{1}$。
* **$j=3$**：$T[3] = \text{'a'}$ 与 $T[\text{next}[3]] = T[1] = \text{'a'}$ **相同**，取 $\text{nextval}[1]$ 得到 $\mathbf{0}$。
* **$j=4$**：$T[4] = \text{'b'}$ 与 $T[\text{next}[4]] = T[2] = \text{'b'}$ **相同**，取 $\text{nextval}[2]$ 得到 $\mathbf{1}$。
* **$j=5$**：$T[5] = \text{'a'}$ 与 $T[\text{next}[5]] = T[3] = \text{'a'}$ **相同**，取 $\text{nextval}[3]$ 得到 $\mathbf{0}$。
* **$j=6$**：$T[6] = \text{'a'}$ 与 $T[\text{next}[6]] = T[4] = \text{'b'}$ **不同**，照抄 $\text{next}[6]$ 得到 $\mathbf{4}$。
* **$j=7$**：$T[7] = \text{'a'}$ 与 $T[\text{next}[7]] = T[2] = \text{'b'}$ **不同**，照抄 $\text{next}[7]$ 得到 $\mathbf{2}$。

## 3. 经典算法实现 (1-indexed)

> 💡 **408 编码说明**：以下 C 语言代码基于王道考研标准。字符串的首位 `S[0]` / `T[0]` 用于存放字符串的长度，实际字符从下标 `1` 开始存储，符合 KMP 手算下标从 1 开始的逻辑。

### 3.1 计算 next 数组
```c
void get_next(char T[], int next[]) {
    int i = 1, j = 0;
    next[1] = 0; // 规定首位为 0
    while (i < T[0]) { // T[0] 存放模式串的长度
        if (j == 0 || T[i] == T[j]) {
            i++;
            j++;
            next[i] = j;
        } else {
            j = next[j]; // 发生失配，j 回溯
        }
    }
}
```

### 3.2 计算 nextval 数组 
```c
void get_nextval(char T[], int nextval[]) {
    int i = 1, j = 0;
    nextval[1] = 0; // 规定首位为 0
    while (i < T[0]) {
        if (j == 0 || T[i] == T[j]) {
            i++;
            j++;
            if (T[i] != T[j]) {
                nextval[i] = j; // 字符不同，照抄 next 数组原本应得的值 (即当前的 j)
            } else {
                nextval[i] = nextval[j]; // 字符相同，直接取回溯位置的 nextval
            }
        } else {
            j = nextval[j]; // 发生失配，j 回溯
        }
    }
}
```

### 3.3 KMP 匹配主函数
```c
int Index_KMP(char S[], char T[], int next[]) {
    int i = 1, j = 1;
    while (i <= S[0] && j <= T[0]) { // S[0] 和 T[0] 分别为主串和模式串的长度
        if (j == 0 || S[i] == T[j]) {
            i++;
            j++; // 匹配成功，指针同步向后移动
        } else {
            j = next[j]; // 匹配失败，模式串指针回溯，主串指针 i 不动
        }
    }
    if (j > T[0]) {
        return i - T[0]; // 匹配成功，返回主串中匹配子串的起始下标
    } else {
        return 0; // 匹配失败
    }
}
```
