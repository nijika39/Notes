# 二叉树与线索二叉树

通过特定的遍历次序，可以将二叉树的非线性结构线性化。线索二叉树则是为了加快查找节点前驱和后继的速度。

## 1. 二叉树的遍历

### 1.1 唯一确定二叉树的条件
* **结论**：仅凭先序、后序和层序序列中的任意两者组合，通常**无法唯一确定**一棵二叉树。必须包含 **[[中序遍历]]** 序列，才能与先序/后序/层序之一结合唯一确定二叉树。

### 1.2 结点遍历先后的充分条件
设 $n, m$ 为一棵二叉树上的两个结点，在以下遍历方式中，$n$ 在 $m$ 前面的充分条件是：
1. **中序遍历**：$n$ 在 $m$ 的左方。
2. **先序遍历**：$n$ 是 $m$ 的祖先。
3. **后序遍历**：$n$ 是 $m$ 的子孙。
* **通用规律**：由于所有遍历方式中一定是“先递归左子树、再递归右子树”，因此当 $n, m$ 分别位于同一个结点的左、右子树上时，$n$ 在任何遍历中一定在 $m$ 前面。

---

### 1.3 特殊二叉树形态

1. **中序相反的树型构造**：
   若有一棵非空二叉树 $T$，要构造一棵与其对应中序序列相反的二叉树 $T'$，只需将 $T$ 从根结点开始，递归地**调换左右子树**（在后序位置执行交换）。
2. **中序序列 = 前序序列，且与后序序列相反**：
   * **形态**：所有结点均无左子树（向右下倾斜的单支树 ↘）。
   * **特点**：结点数 = 树的高度。
3. **中序序列 = 后序序列，且与前序序列相反**：
   * **形态**：所有结点均无右子树（向左下倾斜的单支树 ↙）。
   * **特点**：结点数 = 树的高度。

## 2. 线索二叉树及其遍历

引入 `ltag` 和 `rtag` 标志位，以区分指针指向的是左/右孩子（`tag == 0`）还是前驱/后继线索（`tag == 1`）。

### 2.1 前序线索二叉树

* **第一个结点**：一定是根结点。
* **查询后继结点的规则**：
  1. 若有左孩子（`ltag == 0`），则其**左孩子**为后继。
  2. 若无左孩子但有右孩子（`ltag == 1, rtag == 0`），则其**右孩子**为后继。
  3. 若为叶子结点（`rtag == 1`），则通过线索 `rchild` 直接指向后继。
* **查询前驱结点的规则（基于二叉链表）**：
  1. 若有左线索（`ltag == 1`），则前驱为左孩子指针 `lchild` 指向的结点。
  2. 若无左线索（`ltag == 0`），说明该结点必有左孩子。由于前序遍历的顺序是 **“根 $\to$ 左 $\to$ 右”**，该结点的左右孩子只能是它的后继，它的前驱只可能在其祖先分支中。因此，在**二叉链表存储结构下，无法直接查找到前驱结点**（除非使用带父指针的三叉链表，或者从根结点开始重新进行先序遍历）。

### 2.2 中序线索二叉树

* **第一个结点**：从根结点开始沿着 `lchild` 一直往左走到底的结点（最左下结点）。
* **最后一个结点**：从根结点开始沿着 `rchild` 一直往右走到底的结点（最右下结点）。
* **查询后继结点的规则**：
  * 若 `rtag == 0`，则后继为右子树中的**最左下结点**。
  * 若 `rtag == 1`，则 `rchild` 直接指向后继。
* **查询前驱结点的规则**：
  * 若 `ltag == 0`，则前驱为左子树中的**最右下结点**。
  * 若 `ltag == 1`，则 `lchild` 直接指向前驱。

### 2.3 后序线索二叉树

* **最后一个结点**：一定是根结点。
* **查询前驱结点的规则（基于二叉链表）**：
  1. 若有左线索（`ltag == 1`），则前驱为左孩子指针 `lchild` 指向的结点。
  2. 若无左线索（`ltag == 0`），根据后序遍历 **“左 $\to$ 右 $\to$ 根”** 的次序，根结点的前驱应该是其子树中最后被访问的结点：
     * 若该结点有右孩子（`rtag == 0`），则其前驱为它的**右孩子**。
     * 若该结点无右孩子但有左孩子（`rtag == 1, ltag == 0`），则其前驱为它的**左孩子**。
* **查询后继结点的规则**：
  * 若 `rtag == 1`，则 `rchild` 直接指向后继。
  * 若 `rtag == 0`：
    1. 若其为**根结点**，则无后继。
    2. 若其为双亲的**右孩子**，或其为双亲的**左孩子且双亲无右孩子**，则其后继为**双亲结点**。
    3. 其余情况下，后继需要借助树型判断，在**二叉链表下无法直接查找到后继结点**（需要借助三叉链表中的父指针，或从头开始遍历）。
