# 排序

本章主要介绍各种内部排序算法，包括插入排序、交换排序、选择排序、归并排序、基数排序，以及它们的性能对比。

---

## 1. 插入排序

> [!tip] 提示
> 除特殊说明外，以下插入排序算法中数组的待排序元素起始下标均为 $1$，其中 $S[0]$ 用作哨兵或暂存单元。

### 1.1 直接插入排序

#### 🔑 核心思想
每趟将一个待排序的记录按其关键字大小插入到前面已经排好序的子序列中，直到全部记录插入完成。

#### 📊 性能与特征
* **时间复杂度**：
  * **最好情况**：$O(n)$（初始已有序，每趟只需比较 $1$ 次，移动 $0$ 次）。
  * **最坏情况**：$O(n^2)$（初始逆序）。
  * **平均情况**：$O(n^2)$。
  * **是否与初始序列相关**：**密切相关**（比较和移动次数均取决于初态）。
* **空间复杂度**：$O(1)$（只需常数个辅助单元）。
* **稳定性**：**稳定**。
* **适用性**：顺序存储和链式存储均适用。
* **过程特征**：**不能**在一趟排序中确定任何一个元素的最终位置。

#### ⚠️ 失稳诱因
* **基于顺序存储**：若将 `S[0] < S[j]` 修改为 `S[0] <= S[j]`，则在遇到相同关键字时，待插入元素仍会继续向前移动，从而破坏原相同关键字的相对顺序。
* **基于链式存储**：若将 `pre->next->data <= p->data` 修改为 `< p->data`，则相等结点会插入到已有相同结点的前面，导致不稳定。

#### 💻 核心代码

##### a) 基于顺序存储的直接插入排序
```c
void InsertSort(int *S, int n) {
	int i, j;
	for (i=2; i<=n; i++) {
		S[0] = S[i];                    // S[0]作为哨兵暂存待插入元素
		for (j=i-1; S[0]<S[j]; j--) {   // 只有 < 才能保持排序的稳定性
			S[j+1] = S[j];              // 元素后移
		}
		S[j+1] = S[0];                  // 插入到正确位置
	}
}
```

##### b) 基于链式存储（带头结点）的直接插入排序
```c
void InsertSort(LinkList L) {
    LNode *p = L->next; 
    if (p == NULL || p->next == NULL) {
        return; // 空表或只有一个元素，无需排序
    }

    p = L->next->next;      // p指向待插入的第二个结点
    L->next->next = NULL;   // 构造只含一个结点的有序表

    while (p != NULL) {
        LNode *r = p->next; // 暂存p的后继结点指针
        LNode *pre = L;     // pre用于在有序表中辅助寻找插入位置

        // 在有序表中查找插入位置（寻找第一个数据值大于p->data的结点的前驱，保持稳定性）
        while (pre->next != NULL && pre->next->data <= p->data) {
            pre = pre->next;
        }

        // 将p插入到pre之后
        p->next = pre->next;
        pre->next = p;

        p = r; // 继续处理下一个无序结点
    }
}
```

### 1.2 折半插入排序

#### 🔑 核心思想
对直接插入排序的优化。在已排好序的子序列中利用**折半查找**快速确定插入位置，然后再统一移动元素。

#### 📊 性能与特征
* **时间复杂度**：
  * **最好情况**：$O(n \log_2 n)$ 或 $O(n^2)$（折半查找使比较次数减少为 $O(n \log_2 n)$ 且与初态无关；但移动次数仍为 $O(n^2)$，只有当序列完全有序时移动次数为 $0$。在考研统考中通常归为 $O(n^2)$ 档次）。
  * **最坏情况**：$O(n^2)$。
  * **平均情况**：$O(n^2)$。
  * **是否与初始序列相关**：**部分相关**（比较次数与初始序列状态**无关**，仅与元素个数 $n$ 有关；而元素移动次数仍与初始序列**有关**）。
* **空间复杂度**：$O(1)$。
* **稳定性**：**稳定**。
* **适用性**：**仅适用于顺序存储**（折半查找需要随机存取特性）。
* **过程特征**：**不能**在一趟排序中确定任何一个元素的最终位置。

#### ⚠️ 失稳诱因
若将折半查找中的判断条件 `S[0] < S[mid]` 修改为 `S[0] <= S[mid]`，则当 `S[0] == S[mid]` 时，搜索区间会被迫向左半区移动（即 `high = mid - 1`），导致最后插入到相同关键字的前面，产生失稳。

#### 💻 核心代码
```c
void BiInsertSort(int *S, int n) {
	int low, high, mid;
	for (int i=2; i<=n; i++) {
		S[0] = S[i];
		low = 1, high = i-1;
		while (low <= high) {           // 折半查找插入位置
			mid = (low + high) / 2;
			if (S[0] < S[mid]) {
				high = mid - 1;         // 插入点在左半区
			} else {
				low = mid + 1;          // 插入点在右半区（相等时在右半区找，以保持稳定）
			}
		}
		for (int j=i-1; j>=low; j--) {  // 统一后移
			S[j+1] = S[j];
		}
		S[low] = S[0];                  // 插入
	}
}
```

### 1.3 希尔排序（缩小增量排序）

#### 🔑 核心思想
通过“增量”将待排序序列分割成若干个子序列，分别进行直接插入排序。随着增量逐渐缩小，整个序列呈现“基本有序”，最后进行一次全体直接插入排序。

#### 📊 性能与特征
* **时间复杂度**：
  * **最好情况**：约 $O(n^{1.3})$（取决于增量序列的选择）。
  * **最坏情况**：$O(n^2)$。
  * **平均情况**：约 $O(n^{1.3})$。
  * **是否与初始序列相关**：**密切相关**。
* **空间复杂度**：$O(1)$。
* **稳定性**：**不稳定**（相同关键字可能会被分配到不同的子序列中进行排序，从而改变相对顺序）。
* **适用性**：**仅适用于顺序存储**。
* **过程特征**：**不能**在一趟排序中确定任何一个元素的最终位置。

#### 💻 核心代码
```c
void ShellSort(int *S, int n) {
	int i, j, dk;
	for (dk=n/2; dk>=1; dk/=2) {        // 步长逐步折半减小
		for (i=1+dk; i<=n; i++) {
			S[0] = S[i];                // 暂存待插入元素
			for (j=i-dk; j>0 && S[0]<S[j]; j-=dk) {
				S[j+dk] = S[j];         // 组内元素后移
			}
			S[j+dk] = S[0];
		}
	}
}
```

## 2. 交换排序

### 2.1 冒泡排序

#### 🔑 核心思想
通过相邻两元素的两两比较与交换，让关键字较大的元素像“气泡”一样逐渐向后上浮，每一趟都会将当前未排序部分的最大值（或最小值）交换至其最终位置。

#### 📊 性能与特征
* **时间复杂度**：
  * **最好情况**：$O(n)$（初始已有序，只需进行一趟比较，无需任何交换）。
  * **最坏情况**：$O(n^2)$（初始逆序）。
  * **平均情况**：$O(n^2)$。
  * **是否与初始序列相关**：**密切相关**（比较和交换次数均高度依赖于初始状态）。
* **空间复杂度**：$O(1)$。
* **稳定性**：**稳定**。
* **适用性**：顺序存储和链式存储均适用。
* **过程特征**：**能**确定最终位置。每趟冒泡都会将一个元素放到其最终位置。

#### ⚠️ 失稳诱因
若将相邻比较语句 `S[j-1] > S[j]` 修改为 `S[j-1] >= S[j]`，则当相邻两元素相等时也会进行多余的交换，导致相同元素的相对次序颠倒。

#### ⚠️ 易错点
需要区分**有元素交换**的排序趟数和**总排序趟数**，在实际算法中，还需要一趟无元素交换的排序来提前终止排序算法，因此总排序趟数 = 有元素交换的排序趟数 + 1。

#### 💻 核心代码

##### a) 基于顺序存储的冒泡排序（从后往前扫，最小值上浮至头部）
```c
void BubbleSort(int *S, int n) {
	for (int i=0; i<n-1; i++) {
		int flag = 0;                   // 交换标记
		for (int j=n-1; j>i; j--) {
			if (S[j-1] > S[j]) {        // 只有 > 才能保证排序是稳定的
				swap(S[j-1], S[j]);
				flag = 1;               // 发生交换
			}
		}
		if (!flag) {
			return;                     // 本趟无交换，说明已有序，直接退出
		}
	}
}
```

##### b) 基于链式存储（带头结点）的冒泡排序
```c
void BubbleSort(LinkList L) {
    if (L->next == NULL || L->next->next == NULL) {
        return; 
    }

    LNode *tail = NULL; // 已排好序部分的边界（其后部分均已确定最终位置）
    int swapped = 1;

    while (swapped) {
        swapped = 0;
        LNode *p = L->next; 

        while (p->next != tail) {
            if (p->data > p->next->data) {
                // 仅交换相邻两个结点的数据域，保持链表物理结构不变
                int temp = p->data;
                p->data = p->next->data;
                p->next->data = temp;
                swapped = 1; 
            }
            p = p->next; 
        }
        tail = p; // 本趟结束，当前无序部分的最大值 p 最终定位，更新边界
    }
}
```

### 2.2 快速排序

#### 🔑 核心思想
基于分治法。选择一个枢轴（Pivot）元素，通过一趟划分将序列分为独立的左右两部分，使左半部分均小于枢轴，右半部分均大于枢轴，然后递归对子序列进行上述过程。

#### 📊 性能与特征
* **时间复杂度**：
  * **最好情况**：$O(n \log_2 n)$（每次划分都极其均匀，如每次都在正中间，递归树高度最小）。
  * **最坏情况**：$O(n^2)$（**初始基本有序或完全逆序**，此时退化为冒泡排序，递归树退化为单支树）。
  * **平均情况**：$O(n \log_2 n)$（所有内部排序算法中**平均性能最优**）。
  * **是否与初始序列相关**：**密切相关**（若初始基本有序，快排的效率反而最差）。
* **空间复杂度**：
  * **最好/平均情况**：$O(\log_2 n)$（递归栈深为 $\log_2 n$）。
  * **最坏情况**：$O(n)$（递归树退化，栈深为 $n$）。
* **稳定性**：**不稳定**（划分过程存在跨越式的元素交换）。
* **适用性**：**仅适用于顺序存储**（首尾指针双向逼近需随机存取特性支持）。
* **过程特征**：**能**确定最终位置。每一趟划分都会将当前枢轴（Pivot）元素放到其最终位置。

#### 💻 核心代码
```c
void QuickSort(int *S, int low, int high) {
	if (low < high) {
		int index = Partition(S, low, high); // 划分，获取枢轴最终位置
		QuickSort(S, low, index - 1);       // 递归处理左子表
		QuickSort(S, index + 1, high);      // 递归处理右子表
	}
}

int Partition(int *S, int low, int high) {
	int pivot = S[low];                 // 取第一个元素作为基准
	while (low < high) {
		while (S[high]>=pivot && low<high) {
			high--;                     // 从右往左找第一个小于基准的元素
		}
		S[low] = S[high];               // 移到左边
		while (S[low]<=pivot && low<high) {
			low++;                      // 从左往右找第一个大于基准的元素
		}
		S[high] = S[low];               // 移到右边
	}
	S[low] = pivot;                     // 枢轴元素归位
	return low;                         // 返回枢轴位置
}
```

## 3. 选择排序

### 3.1 简单选择排序

#### 🔑 核心思想
每一趟在后面未排序的子序列中通过比较直接挑选出关键字最小的元素，将其与未排序子序列的第一个元素交换。

#### 📊 性能与特征
* **时间复杂度**：
  * 最好、最坏、平均均为 **$O(n^2)$**（无论初始状态如何，第 $i$ 趟都要在后续 $n-i$ 个元素中扫一遍以寻最小值，比较次数始终为 $n(n-1)/2$ ）。
  * **是否与初始序列相关**：比较次数与初态**无关**，但元素移动次数与初态**有关**（若已有序则移动 $0$ 次）。
* **空间复杂度**：$O(1)$。
* **稳定性**：**不稳定**（如序列 `[5, 5, 2]`，第一趟会将第一个 `5` 与 `2` 交换，从而改变了两个 `5` 的相对位置）。
* **适用性**：顺序存储和链式存储均适用。
* **过程特征**：**能**确定最终位置。每趟简单选择排序都会将当前无序区中的最小（或最大）元素交换到它的最终位置。

#### 💻 核心代码

##### a) 基于顺序存储的简单选择排序
```c
void SelectSort(int *S, int n) {
	for (int i=0; i<n-1; i++) {
		int min = i;
		for (int j=i+1; j<n; j++) {
			if (S[j] < S[min]) {
				min = j;                // 记录本趟最小值的下标
			}
		}
		if (min != i) {
			swap(S[i], S[min]);         // 交换至当前未排子表头部
		}
	}
}
```

##### b) 基于链式存储的简单选择排序
```c
void SelectSort(Node *head) {
    if (head == NULL || head->next == NULL) {
        return;
    }
    // p 从第一个有效节点开始往后扫
    for (Node *p = head->next; p != NULL; p = p->next) {
        Node* min = p;
        // q 从 p 的下一个节点开始寻找最小值
        for (Node *q = p->next; q != NULL; q = q->next) {
            if (q->x < min->x) {
                min = q;
            }
        }
        // 找到最小值后，仅交换数据域，不改变物理指针结构
        if (min != p) {
            int temp = p->x;
            p->x = min->x;
            min->x = temp;
        }
    }
}
```

### 3.2 堆排序

#### 🔑 核心思想
建立在大根堆（或小根堆）物理存储基础上的选择排序。大根堆的根结点存最大值，将其与堆底最后一个元素交换后，将剩余元素重新调整（Heapify）为大根堆，以此循环实现排序。

#### 📊 性能与特征
* **时间复杂度**：
  * 最好、最坏、平均均为 **$O(n \log_2 n)$**。建堆的时间复杂度为 $O(n)$，每次调整的时间复杂度为 $O(\log_2 n)$，共调整 $n-1$ 次。
  * **是否与初始序列相关**：**无关**（不论初始序列状态如何，堆调整和建堆的过程均需执行固定的二叉树下沉和比较操作）。
* **空间复杂度**：
  * **迭代版**：$O(1)$。
  * **递归版**：$O(\log_2 n)$（递归栈深度）。
* **稳定性**：**不稳定**（堆调整过程中，跨层级、跨分支的元素交换容易破坏原本相等的关键字相对位置）。
* **适用性**：**仅适用于顺序存储**（因为需要通过下标 `2i` 和 `2i+1` 快速寻址双亲结点的孩子结点，链式结构极难实现此定位）。
* **过程特征**：**能**确定最终位置。每趟将堆顶（最大值）交换至堆底末尾，代表此元素已确定其最终位置。

#### 💻 核心代码

##### 3.2.1 建立最大堆
```c
void BuildMaxHeap(int *S, int len) {
	// 从非叶子节点 len/2 向下调整直至根节点 1
	for (int i=len/2; i>=1; i--) {      // 修正：必须是 i>=1，漏掉1会导致无法调整到根节点
		Heapify(S, i, len);
	}
}
```

##### 3.2.2 Heapify函数（下沉操作）

###### a) 递归版
```c
void Heapify(int *S, int i, int len) {
	int largest = i;
	int left = 2*i, right = 2*i+1;
	
	if (left <= len && S[left] > S[largest]) largest = left;   // 修正：对比的是元素值，而非下标
	if (right <= len && S[right] > S[largest]) largest = right; // 修正：同上
	
	if (largest != i) {
		swap(S[i], S[largest]);
		Heapify(S, largest, len);       // 递归向下调整
	}
}
```

###### b) 迭代版
```c
void Heapify(int *S, int i, int len) {
	S[0] = S[i];                        // S[0] 暂存根节点值
	for (int k=2*i; k<=len; k*=2) {     // k自当前结点的左孩子开始
		if (k < len && S[k] < S[k+1]) {
			k++;                        // k指向左右孩子中较大者
		}
		if (S[0] >= S[k]) {
			break;                      // 根节点值大于等于左右孩子的最大值，无需下沉
		} else {
			S[i] = S[k];                // 较大孩子结点值上移至双亲节点
			i = k;                      // 更新双亲节点下标，以便继续向下筛选
		}
	}
	S[i] = S[0];                        // 将原根节点值放入其最终下沉的位置
}
```

##### 3.2.3 堆排序主函数
```c
void HeapSort(int *S, int len) {
	BuildMaxHeap(S, len);               // 初始建堆
	for(int i=len; i>1; i--) {
		swap(S[i], S[1]);               // 将堆顶元素（当前最大）与当前堆底交换，确定最终位置
		Heapify(S, 1, i-1);             // 对剩余 i-1 个元素进行调整，使其重新成为最大堆
	}
}
```

##### 3.2.4 最大堆插入（上浮操作）
```c
void SiftUp(int *S, int i) {
	int temp = S[i];                    // 暂存待上浮元素
	while (i > 1 && S[i/2] < temp) {    // 如果双亲节点小于待插入元素
		S[i] = S[i/2];                  // 物理位置：双亲下沉，留出空位
		i = i / 2;                      // 逻辑位置：待上浮结点上移
	}
	S[i] = temp;                        // 放入最终上浮位置
}

void MaxHeapInsert(int *S, int n, int x) {
	S[++n] = x;                         // 插入到堆底末尾
	SiftUp(S, n);                       // 执行上浮操作
}
```

##### 3.2.5 Top-K问题
求一个海量数组 $S$ 中最大的 $K$ 个值，可使用容量为 $K$ 的**最小堆**进行过滤。
* **时间复杂度**：$O(n \log_2 k)$（$n$ 为原数组长度）。

```c
int* TopK(int* S, int n, int k) {
	int* T = (int *)malloc(sizeof(int) * (k+1));
	for (int i=1; i<=k; i++) {
		T[i] = S[i];                    // 将S的前K个元素复制到堆中
	}
	BuildMinHeap(T, k);                 // 建立容量为K的最小堆
	for (int i=k+1; i<=n; i++) {
		if (S[i] <= T[1]) continue;     // 小于或等于堆顶则丢弃（大者才能入选）
		else {
			T[1] = S[i];                // 大者替换堆顶
			HeapifyMin(T, 1, k);        // 重新向下调整为最小堆
		}
	}
	return T;                           // 堆内保留的K个元素即为最大的K个值
}
```

## 4. 归并排序

#### 🔑 核心思想
基于分治和“合并”。将含有 $n$ 个记录的初始序列看成 $n$ 个有序子序列。两两合并（2路归并），得到 $\lceil n/2 \rceil$ 个长度为 $2$ 的子序列；然后再两两合并……如此重复，直至合并成一个长度为 $n$ 的有序序列。

#### 📊 性能与特征
* **时间复杂度**：
  * 最好、最坏、平均均为 **$O(n \log_2 n)$**。共需进行 $\lceil \log_2 n \rceil$ 趟归并，每趟归并的时间复杂度为 $O(n)$。
  * **是否与初始序列相关**：**无关**（归并的递归树结构与初态无关，均必须无条件拆分并两两合并）。
* **空间复杂度**：**$O(n)$**（合并操作时，需要一个与待排数组同等容量的辅助数组 $B$）。
* **稳定性**：**稳定**。
* **适用性**：顺序存储和链式存储均适用。
* **过程特征**：**不能**在一趟排序中确定任何一个元素的最终位置（仅在子序列内实现有序，局部有序并不代表全局位置定位，只有在最后一趟合并结束时才能完全定位）。

#### ⚠️ 失稳诱因
在合并操作 `Merge` 的选择元素时，逻辑判断为 `if (B[i] <= B[j])`。如果将其修改为 `B[i] < B[j]`，则当两个子序列中当前指向的两个元素相等时，会优先将右半部分的 `B[j]` 放入新表，破坏了原本相同元素的相对次序，引发失稳。

#### 💻 核心代码
```c
void Merge(int *A, int *B, int low, int mid, int high) {
    for (int i = low; i <= high; i++) {
        B[i] = A[i];                    // 将 A 的内容复制到辅助数组 B 中
    }

    int i = low, j = mid + 1, k = low;

    // 两两合并，注意 <= 是保证归并稳定的核心
    for (; i <= mid && j <= high; k++) {
        if (B[i] <= B[j]) {
            A[k] = B[i++];
        } else {
            A[k] = B[j++];
        }
    }

    while (i <= mid)  A[k++] = B[i++];  // 若左半区未合并完，复制剩余元素
    while (j <= high) A[k++] = B[j++];  // 若右半区未合并完，复制剩余元素
}

void MergeSort(int *A, int *B, int low, int high) {
    if (low < high) {
        int mid = low + (high - low) / 2; // 防溢出的中值写法

        MergeSort(A, B, low, mid);       // 递归划分左子表
        MergeSort(A, B, mid + 1, high);  // 递归划分右子表

        if (A[mid] > A[mid + 1]) {
            Merge(A, B, low, mid, high); // 只有在不满足局部有序时才合并（小幅优化）
        }
    }
}
```

## 5. 基数排序（Radix Sort）

#### 🔑 核心思想
非比较型排序算法。通过多关键字的“分配”和“收集”来实现排序。一般基于链式队列进行。按从低位到高位（LSD）或高位到低位（MSD）的顺序，将元素的各个数位（关键字）依次送入代表不同数位的“桶”队列中（分配），再依次按队列出队并重新连接（收集）。

#### 📊 性能与特征
* **时间复杂度**：
  * 最好、最坏、平均均为 **$O(d(n+r))$**。其中 $d$ 为关键字位数（即分配与收集的趟数），$r$ 为基数（桶的个数，例如十进制为 $10$），$n$ 为元素个数。一趟分配耗时 $O(n)$，收集耗时 $O(r)$。
  * **是否与初始序列相关**：**无关**。
* **空间复杂度**：**$O(r)$**（需要 $r$ 个队列的头、尾指针）。
* **稳定性**：**稳定**（因为分配和收集时均使用了 FIFO 的队列存储，保证了相对相对位置不变）。
* **适用性**：顺序存储和链式存储均可（常用单链表结合链队列实现桶，分配和收集时仅需修改结点指针，极度高效）。
* **过程特征**：**不能**在一趟排序中确定任何一个元素的最终位置（仅在经历全部 $d$ 趟分配与收集之后，所有数据才真正排列在最终位置上）。

| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **直接插入排序** | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | **是** | 否 | 否 |
| **折半插入排序** | $O(n \log_2 n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | **是** | **是** | 否 |
| **冒泡排序** | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | **是** | 否 | **是** |
| **简单选择排序** | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 否 | 否 | **是** |
| **希尔排序** | - | $O(n^{1.3})$ | $O(n^2)$ | $O(1)$ | 否 | **是** | 否 |
| **快速排序** | $O(n \log_2 n)$ | $O(n \log_2 n)$ | $O(n^2)$ | $O(\log_2 n)$ | 否 | **是** | **是** |

> [!key] 考研核心记忆
> * **不稳定**：快（快速排序）、些（希尔排序）、选（简单选择）、堆（堆排序）
> * **一趟可定位**：冒泡排序、快速排序、简单选择排序、堆排序 
> * **对空间要求最高**：归并排序（$O(n)$）、快速排序（$O(\log_2 n)$）、基数排序（$O(r)$）。
> * **时间复杂度与初态完全无关**：简单选择排序、堆排序、归并排序、基数排序
