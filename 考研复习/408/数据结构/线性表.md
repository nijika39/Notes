# 线性表

本篇笔记整理了线性表在 408 考研中的核心易错概念，以及高频考查的代码优化技巧。

---

## 1. 易错点

### 1.1 静态分配与动态分配
* **一维数组的双重属性**：一维数组既可以静态分配，也可以动态分配。
* **扩容的本质**：动态分配时的“扩容”过程与 `malloc` 本身无关。`malloc` 的职责仅是从**堆（Heap）**中分配一段连续的存储空间。
* **扩容的步骤**：当空间占满时，需要程序员编写代码**手动开辟另一块更大的连续空间**，然后将原数据复制过去，最后释放原空间，以此实现动态扩容。
* **🔑 重点**：动态分配的线性表**并不是**链式存储，它同样属于**顺序存储结构**。

### 1.2 静态链表 (Static Link List)
* **实现原理**：**[[静态链表]]**是用静态数组实现的，其长度在声明时确定，**不能改变**（即不能动态扩容）。
* **存储与访问**：虽然静态链表的物理存储空间是连续分配的，但它**不支持随机访问**（仍需通过游标 `next` 挨个遍历）。
* **为什么不能动态扩容？** 静态链表设计的初衷是为了在**没有指针的低级语言**中提供链表的实现。既然没有指针，自然无法在运行时动态分配内存。
* **结构体定义 (C 语言)**：
```c
#define MaxSize 100
typedef int ElemType;

typedef struct {
    ElemType data;
    int next; // 游标，代表下一个结点的数组下标
} SLinkList[MaxSize];
```

### 1.3 逻辑结构视角下的“数组” (Array)
* **逻辑定义**：数组是**线性表的推广**，属于一种**逻辑结构**（如二维数组可看作是每个元素也为定长线性表的线性表）。
* **操作局限**：由于数组的**维数和维界在创建时就已确定**，因此其抽象数据类型（ADT）中**没有规定插入和删除的基本操作**，也**不能动态修改大小**（只允许进行“读”和“改”操作）。
* **双重存储**：数组作为一种逻辑结构，既可以使用**顺序存储**，也可以使用**链式存储**（如稀疏矩阵的十字链表实现）。

### 1.4 考研高频存储归类与误区
* **考研链式存储家族**：以下三者在考研大纲中均属于**链式存储结构**：
  * **单/双/循环链表**：最经典的链式存储。
  * **十字链表**：稀疏矩阵（多维数组）的**链式存储**表达。其物理内存占用虽能动态增减，但其逻辑维界（网格行列数）依然固定。
  * **静态链表**：**链式存储**（物理上虽借用数组分配连续空间，但节点间的逻辑关系依靠游标链接，不支持随机访问，常作为选择题分类干扰项）。

## 2. 代码技巧

### 2.1 逆置顺序表 (Array Reverse)
* **代码实现**：
```c
void Reverse(int R[], int a, int b) {
    // 逆置范围为 [a, b] 的子数组
    for (int i = 0; i < (b - a + 1) / 2; i++) {
        int temp = R[a + i];
        R[a + i] = R[b - i];
        R[b - i] = temp;
    }
}
```
> 💡 **原理分析**：
> * 对于**长度为奇数**的顺序表，$\frac{b-a+1}{2}$ 正好指向中间元素。
> * 对于**长度为偶数**的顺序表，$\frac{b-a+1}{2}$ 指向右半部分数组的首个元素。
> * 综上，设置循环条件为 `i < (b-a+1)/2` 即可精确完成左右对调而不越界。

### 2.2 $O(1)$ 时间复杂度前插与删除单链表结点
若要求在给定结点前插入新结点，或者删除给定结点，在允许修改数据域（`data`）的前提下，可以通过“数据交换”将时间复杂度降低至 $O(1)$。

##### A. 前插技巧（在给定结点 `*p` 之前插入新结点 `*s`）
```c
// 1. 先将 s 插入到 p 的后面（后插法）
s->next = p->next;
p->next = s;
// 2. 交换 p 和 s 的数据域，间接实现前插
ElemType temp = p->data;
p->data = s->data;
s->data = temp;
```

##### B. 删除技巧（删除给定结点 `*p`）
> 🎭 **形象比喻**：此方法本质是 **“夺舍”** —— 抢走后继结点的灵魂（数据），然后消灭后继结点的肉体（空间）。
```c
LNode *q = p->next;     // q 指向 p 的后继结点
p->data = q->data;     // 把后继结点的数据“夺舍”过来覆盖 p 的数据
p->next = q->next;     // 绕过 q，指向后继的后继
free(q);               // 释放 q（原本后继结点的肉体）
```

### 2.3 逆置单链表（头插法逆置段）
* **基本思想**：找到待逆置段的前一个节点设为 `p`（其行为类似于一个链表的头结点），先断开它与待逆置段的连接，随后通过**头插法**将待逆置元素依次插入。
```c
// 初始化：断开连接，q 指向待逆置段的首结点
LNode *q = p->next;
p->next = NULL; 

LNode *r;
while (q != NULL) {
    r = q->next;       // 暂存后继，防止断链
    q->next = p->next; // 头插法插入到 p 的后面
    p->next = q;
    q = r;             // 指针后移
}
```

### 2.4 归并两个有序链表

#### a) 递增归并（尾插法）

```c
// 假设 A 和 B 是带头结点的递增单链表，合并后 C 也是递增的
LinkList MergeList_Ascending(LinkList A, LinkList B) {
    LNode* p = A->next;  // 遍历 A 的指针
    LNode* q = B->next;  // 遍历 B 的指针
    LNode* C = A;        // 复用 A 的头结点作为 C 的头结点
    LNode* r = C;        // r 为 C 的尾指针，用于尾插法
    
    while (p != NULL && q != NULL) {
        if (p->data <= q->data) {
            r->next = p;
            r = p;
            p = p->next;
        } else {
            r->next = q;
            r = q;
            q = q->next;
        }
    }
    
    // 将剩余的链表部分直接链接到尾部（因为本来就是有序的）
    if (p != NULL) r->next = p;
    else r->next = q;
    
    free(B); // B 的头结点不需要了，释放掉
    return C;
}
```

#### b) 逆序归并（头插法）

```c
// 递增单链表 A 和 B 合并为一个递减单链表（头插法）
LinkList MergeList_Descending(LinkList A, LinkList B) {
    LNode* p = A->next;
    LNode* q = B->next;
    LNode* next;
    A->next = NULL; // 将 A 的头结点作为结果链表的头结点，并初始化为空
    
    while (p != NULL && q != NULL) {
        if (p->data <= q->data) {
            next = p->next;     // 暂存 p 的下一个节点
            p->next = A->next;  // 将 p 头插法插入到 A 之后
            A->next = p;
            p = next;           // 恢复 p 指针
        } else {
            next = q->next;
            q->next = A->next;
            A->next = q;
            q = next;
        }
    }
    
    // 如果 A 没遍历完，必须【逐个】头插，才能将剩余递增序列逆序
    while (p != NULL) {
        next = p->next;
        p->next = A->next;
        A->next = p;
        p = next;
    }
    
    // 如果 B 没遍历完，同理必须【逐个】头插
    while (q != NULL) {
        next = q->next;
        q->next = A->next;
        A->next = q;
        q = next;
    }
    
    free(B); // 释放 B 的头结点
    return A;
}
```
