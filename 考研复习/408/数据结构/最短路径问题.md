# 最短路径问题

本章主要介绍最短路径问题的两个经典算法：Dijkstra 算法和 Floyd 算法。

---

## 一、 Dijkstra 算法（迪杰斯特拉算法）

### 1. 算法步骤
用于解决**单源最短路径**问题（求图中某一顶点到其余所有顶点的最短路径）。

1. **初始化**：
   * 引入辅助数组 `dist[]`（存储当前源点到各顶点的最短路径长度）和 `path[]`（记录路径上当前顶点的前驱）。
   * 集合 $S$ 用于存放已求得最短路径的顶点，初始时 $S = \{v_0\}$。
2. **贪心选择**：
   * 从未并入集合 $S$ 的顶点中，选出一个顶点 $u$，使得 `dist[u]` 的值最小。将 $u$ 加入集合 $S$。
3. **更新（松弛操作）**：
   * 以顶点 $u$ 为中转点，更新源点到其余未加入 $S$ 的顶点 $j$ 的距离。若 `dist[u] + Edge[u][j] < dist[j]`，则更新 `dist[j] = dist[u] + Edge[u][j]`，并将 `path[j]` 设为 $u$。
4. **循环**：
   * 重复步骤 2、3，共执行 $n-1$ 次，直到所有顶点都加入集合 $S$。

---

### 2. C 语言代码实现（基于邻接矩阵）
```c
#define MaxVertexNum 100
#define INF 0x3f3f3f3f

typedef struct {
    char vex[MaxVertexNum];
    int edge[MaxVertexNum][MaxVertexNum];
    int vexnum, arcnum;
} MGraph;

// Dijkstra 算法
void Dijkstra(MGraph G, int v, int dist[], int path[]) {
    int s[MaxVertexNum]; // 标记是否已求得最短路径（1表示在S集中，0表示不在）
    
    // 初始化
    for (int i = 0; i < G.vexnum; i++) {
        dist[i] = G.edge[v][i];
        s[i] = 0;
        if (G.edge[v][i] < INF && i != v) {
            path[i] = v; // 记录前驱
        } else {
            path[i] = -1;
        }
    }
    s[v] = 1; 
    dist[v] = 0; // 源点自身初始化

    // 迭代 n-1 次
    for (int i = 1; i < G.vexnum; i++) {
        int min = INF, u = -1;
        
        // 1. 寻找当前未加入 S 且 dist 最小的顶点 u
        for (int j = 0; j < G.vexnum; j++) {
            if (s[j] == 0 && dist[j] < min) {
                min = dist[j];
                u = j;
            }
        }
        
        if (u == -1) break; // 若其余顶点均不可达，提前结束
        s[u] = 1; // 将 u 并入集合 S
        
        // 2. 更新从源点 v 经过 u 达到其余顶点的最短路径长度
        for (int j = 0; j < G.vexnum; j++) {
            if (s[j] == 0 && G.edge[u][j] < INF) {
                if (dist[u] + G.edge[u][j] < dist[j]) {
                    dist[j] = dist[u] + G.edge[u][j];
                    path[j] = u; // 更改前驱顶点
                }
            }
        }
    }
}
```

---

### 3. Dijkstra 路径输出方法
在 Dijkstra 算法结束后，`path[]` 数组中存储的是当前顶点在最短路径上的**直接前驱**。我们可以通过**递归（回溯）**从终点往前寻找前驱，从而顺序输出完整的源点到终点的路径。

```c
// 递归输出从源点到顶点 u 的最短路径（顺序输出）
void PrintDijkstraPath(int path[], int u) {
    if (path[u] == -1) {
        printf("%d", u); // 递归到起点（没有前驱的结点），开始输出起点
        return;
    }
    PrintDijkstraPath(path, path[u]); // 先递归打印前驱路径
    printf(" -> %d", u);              // 再打印当前结点
}
```

## 二、 Floyd 算法（弗洛伊德算法）

### 1. 算法步骤
用于解决**多源最短路径**问题（求任意两个顶点之间的最短路径）。

1. **初始化**：
   * 定义距离矩阵 $A$（初始化为邻接矩阵）和中转点路径矩阵 $path$（初始化为 -1）。
2. **核心动态规划过程**：
   * 依次试探以顶点 $k$ ($0 \le k < n$) 作为中间过渡点。
   * 对于每一对顶点 $(i, j)$，判断是否通过 $k$ 中转后距离变短。
   * **转移方程**：若 $A[i][k] + A[k][j] < A[i][j]$，则更新：
     $$A[i][j] = A[i][k] + A[k][j]$$
     并记录 $path[i][j] = k$。
3. **循环迭代**：
   * 必须依次把所有顶点 $k$ 作为最外层循环，迭代完 $n$ 轮后，距离矩阵 $A$ 即为最终的任意两点最短路径长度。

---

### 2. C 语言代码实现
```c
#define MaxVertexNum 100
#define INF 0x3f3f3f3f

void Floyd(MGraph G, int A[][MaxVertexNum], int path[][MaxVertexNum]) {
    // 1. 初始化矩阵 A 和 path
    for (int i = 0; i < G.vexnum; i++) {
        for (int j = 0; j < G.vexnum; j++) {
            A[i][j] = G.edge[i][j];
            path[i][j] = -1; // 默认不经过任何中转点
        }
    }
    
    // 2. 三重循环核心（注意：中转点 k 必须在外层！）
    for (int k = 0; k < G.vexnum; k++) {
        for (int i = 0; i < G.vexnum; i++) {
            for (int j = 0; j < G.vexnum; j++) {
                if (A[i][k] < INF && A[k][j] < INF) { // 防止相加时整数溢出
                    if (A[i][k] + A[k][j] < A[i][j]) {
                        A[i][j] = A[i][k] + A[k][j];
                        path[i][j] = k; // 记录中转点
                    }
                }
            }
        }
    }
}
```

---

### 3. Floyd 路径输出方法
在上述实现中，`path[i][j] = k` 采用的是**“中转点记录法”**。输出顶点 $i$ 到 $j$ 的完整路径必须通过**递归**将路径中转点展开。

```c
// 递归打印 i 到 j 之间的所有“中间中转点”（不含起点和终点）
void printPath(int i, int j, int path[][MAXV]) {
    int k = path[i][j];
    if (k == -1) {
        return; // -1 表示 i 和 j 之间已经没有更深的中转点，递归出口
    }
    printPath(i, k, path); // 1. 递归打印左半边：i 到 k 的中转路径
    printf("%d ", k);      // 2. 打印当前的中转点 k
    printPath(k, j, path); // 3. 递归打印右半边：k 到 j 的中转路径
}

// 打印自起点 i 至终点 j 的完整顺序路径
void showFullRoute(int i, int j, int path[][MAXV]) {
    printf("%d ", i);       // 1. 打印起点
    printPath(i, j, path);  // 2. 递归打印中间所有中转点
    printf("%d\n", j);      // 3. 打印终点
}
```

### 4. Floyd 算法检测图中的负权回路（负环）

> [!important] 检测原理（408 重难点选择题考点）
> **Floyd 算法可以有效检测图中是否存在负权回路（负环）。**

#### 检测原理
1. 顶点到自身的距离矩阵 $A[i][i]$ 初始时设置为 `0`。
2. 当图中存在一条包含顶点 $i$ 且权值之和为负数的环路时，意味着从 $i$ 出发绕一圈回到 $i$ 的成本小于 `0`。
3. Floyd 算法的三重循环会尝试所有的路径组合。在考查到经过中转点 $k$ 时，必然会出现：
   $$A[i][k] + A[k][i] < A[i][i]$$
   即：
   $$A[i][k] + A[k][i] < 0$$
4. 一旦发生此更新，对角线上的 $A[i][i]$ 将被更新为一个**小于 0 的负数**（同时 `path[i][i]` 将被修改为中转点 $k$ 而不再是初始的 `-1`）。

#### 校验方法
算法运行结束后，遍历对角线元素。若存在一个顶点满足 $A[i][i] < 0$（或者 `path[i][i] != -1`），即可断定该图中**必定存在包含顶点 $i$ 的负权回路**。

## 三、 Dijkstra 算法和 Floyd 算法对比

在 408 备考中，两者的对比是选择题的热门考点：

| | BFS 算法 | Dijkstra 算法 | Floyd 算法 |
| :--- | :--- | :--- | :--- |
| **用途** | 求单源最短路径 | 求单源最短路径 | 求各顶点之间的最短路径 |
| **无权图** | 适用 | 适用 | 适用 |
| **带权图** | 不适用 | 适用 | 适用 |
| **带负权值的图** | 不适用 | 不适用 | 适用 |
| **带负权回路的图** | 不适用 | 不适用 | 不适用 |
| **时间复杂度** | $O(\lvert V \rvert^2)$ 或 $O(\lvert V \rvert + \lvert E \rvert)$ | $O(\lvert V \rvert^2)$ | $O(\lvert V \rvert^3)$ |

堆优化版本的 Dijkstra 算法时间复杂度为 $O(\lvert E \rvert \log \lvert V \rvert)$，使用最小堆实现

## 四、 知识点与选择题避坑指南

> [!important] 考研必背干货
> 1. **[[BFS]] 与最短路径**：
>    * BFS 确实可以求解最短路径，但**仅限**于求解**非带权图**（或者所有边权值均相等的图）的单源最短路径。
> 2. **最短路径的路径特征**：
>    * 图中如果不存在负权回路，则任意两个顶点之间的最短路径**一定是[[简单路径]]**（不包含重复结点的环路）。
> 3. **Dijkstra 为什么不能处理负权边？（选择题常考原因）**：
>    * 因为 Dijkstra 属于贪心算法，其核心假设是“一旦某个顶点并入集合 $S$，它的最短路径距离就确定了，后续不会再被修改”。但如果存在负权边，通过后续未并入顶点作为中转，可能会产生更短的路径，这破坏了贪心选择的正确性。
> 4. **路径的还原方法**：
>    * **Dijkstra** 还原：通过 `path[]` 数组，从终点往前倒推直接前驱。
>    * **Floyd** 还原：若 `path[i][j] = k`，说明最短路径需要经过中转点 $k$，需要递归拆分求解 $i \to k$ 和 $k \to j$。
