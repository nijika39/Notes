## 1. 摩尔投票算法 (Moore's Voting Algorithm)

### 📌 题目背景
> **408 统考真题（2013年大题 41 题改编）**
> **题目描述**：已知一个长度为 $n$ 的整数序列，若其中存在一个数字出现的次数超过了数组长度的一半（即 $> n/2$），则称该数字为**主元素**。请设计一个在时间和空间两方面都尽可能高效的算法，找出该主元素。若不存在主元素，则返回 -1。

### 💡 核心思想（摩尔投票法）
这是一种典型的“同归于尽”的思想。
1. **第一阶段（对抗阶段）**：
   * 设置一个候选人 `num` 和其对应的票数 `count`。
   * 顺序扫描数组。若当前数字与候选人 `num` 相同，则票数加 1；若不同，则票数减 1（代表两个不同的数相互抵消）。
   * 若 `count` 减为 0，说明之前的数已经全部两两抵消，我们将下一个遇到的数字设为新的候选人，并将票数重置为 1。
   * **原理**：由于主元素的个数超过了一半，即便它与其他所有非主元素一一抵消，最终剩下的候选人也必定是它。
2. **第二阶段（检验阶段）**：
   * 摩尔投票法只能保证“如果存在主元素，它一定是最终存留在 `num` 中的数”。但如果数组本来就没有主元素（例如 `[1, 2, 3]`），它也会剩下一个无辜的候选人。
   * 因此，我们需要进行第二轮扫描，统计该候选人在数组中实际出现的总次数。若确实 $> n/2$，则确认它是主元素；否则返回 -1。

### 📊 复杂度分析
* **时间复杂度**：$O(n)$，只需两遍线性扫描。
* **空间复杂度**：$O(1)$，仅需常数个辅助变量，无需开辟额外空间。

---

### 💻 算法实现 (C 语言)

```c
#include <stdio.h>

/**
 * @brief 寻找数组中的主元素（出现次数超过一半的数）
 * @param A 目标整数数组
 * @param n 数组长度
 * @return int 若存在主元素则返回该元素值，否则返回 -1
 */
int findMajorityElement(int A[], int n) {
    if (n <= 0) return -1;

    int num = A[0]; // 初始化候选人
    int count = 1;  // 候选人票数

    // 第一阶段：确定候选人
    for (int i = 1; i < n; i++) {
        if (count == 0) {
            num = A[i];
            count = 1;
        } else if (A[i] == num) {
            count++;
        } else {
            count--;
        }
    }

    // 第二阶段：验证候选人是否真的超过了一半
    int actual_count = 0;
    for (int i = 0; i < n; i++) {
        if (A[i] == num) {
            actual_count++;
        }
    }

    // 判断实际出现次数是否大于 n / 2
    if (actual_count > n / 2) {
        return num;
    } else {
        return -1; // 不存在主元素，返回 -1
    }
}
