# 外部排序

外部排序主要包含两阶段：**① 生成初始归并段（通常由内存内部排序完成）** $\to$ **② 多路平衡归并（通过磁盘I/O一趟趟读写完成）**。由于磁盘 I/O 速度远慢于内存，算法优化的核心目的在于**极力减少磁盘 I/O 次数**。

---

## 1. 核心缓冲区与关系模型

为了在内存中处理庞大的磁盘文件，内存工作区、输入/输出缓冲区、磁盘物理块、归并段的大小关系如下：

* **磁盘物理块（磁盘块）**：外存读写的**最小基本单位**。设其容量为 $B$ 个记录。
* **输入/输出缓冲区**：内存中分配的临时中转站。在归并阶段，**每个缓冲区的大小通常恰好等于 1 个磁盘物理块（$B$ 个记录）**。
* **内存工作区**：在 $k$ 路归并时，内存工作区必须被拆分为：
  $$\text{内存工作区} = k \text{ 个输入/输出缓冲区} + 1 \text{ 个输出缓冲区} = (k + 1) \text{ 个磁盘物理块}$$
* **初始归并段**：其大小 $L$ 在常规内部排序时，等于内存工作区的总容量（即 $L = (k+1) \times B$）。
* **后续归并段**：随着归并的进行，归并段会迅速变大（首趟归并后变为 $k \times L$，次趟 $k^2 \times L$ ），**内存一次性根本装不完**，必须依靠输入/输出缓冲区以“物理块”为单位分批读入、合并、写出。

## 2. 置换-选择排序（优化阶段一：让初始段更长）

#### 🔑 核心目的
通过在内存中建立一个大小为 $M$ 的缓冲区，产生**长度远大于 $M$ 的初始归并段**（平均长度可达 $2M$）。初始段变长 $\to$ 初始归并段个数 $r$ 变少 $\to$ 归并树高度降低 $\to$ 磁盘 I/O 次数大幅减少。

#### 💻 运行机理

设初始待排文件为 **FI**，归并段输出文件为 **FO**，内存工作区为 **WA**（容量为 $w$ 个记录，初始为空）。当前已写入 FO 的记录（即当前归并段中已输出的最大值）记为 **MINIMAX**。

1. **初始化填充**：从 **FI** 读入 $w$ 个记录，填满整个工作区 **WA**。
2. **决出段首元素**：从 **WA** 中挑选出**关键字最小**的记录，记为当前 **MINIMAX**，并将其写入 **FO**（当前归并段的起点）。
3. **填补与迭代**：
   * **填补**：若 **FI** 非空，立即从 **FI** 读入下一个新记录填补 **WA** 的空位。
   * **筛选**：在 **WA** 中，过滤出所有**关键字 $\ge$ 当前 MINIMAX** 的记录（即有资格放入当前递增段的记录）。
   * **输出**：在这些候选记录中，选择**关键字最小者**作为新的 **MINIMAX** 写入 **FO**。
   * **循环**：重复上述“填补 $\to$ 筛选 $\to$ 输出”过程。
1. **截断完成分段**：直到 **WA** 中所有记录的关键字都 $<$ 当前 **MINIMAX** 时，说明当前初始归并段无法继续延长，此段输出结束。
2. **开启新归并段**：重新执行步骤 2 ~ 4（即在 WA 的剩余记录中重新选一个绝对最小值作为新起点，开始输出下一个初始归并段）。
3. **终止判定**：如此反复，直到 **FI** 与 **WA** 均为空，所有初始归并段生成完毕。

## 3. 败者树（优化阶段二：降低内存归并的时间开销）

#### 🔑 核心目的
在进行 $k$ 路归并时，原本在 $k$ 个元素中选出最小者需要进行 $k-1$ 次比较。引入败者树后，每次调整只需进行 $\lceil \log_2 k \rceil$ 次比较，**使内部归并的时间复杂度与归并路数 $k$ 无关**。

#### 📊 败者树的结构性质
* **性质 1（高度公式）**：
  败者树的核心是一棵含有 $k$ 个叶子结点的完全二叉树。其高度 $H$（**包含叶子结点层，不包含附加结点 `ls[0]`**）为：
  $$H = \lceil \log_2 k \rceil + 1$$
* **性质 2（建树比较次数）**：
  初始构建一棵 $k$ 路败者树，决出第一个全局冠军，共需要进行 **$k-1$ 次** 比较。
* **性质 3（调整比较次数）**：
  输出冠军后，补充新元素进入叶子结点，自底向上调整败者树决出新冠军，最多需要进行 **$\lceil \log_2 k \rceil$ 次** 比较（即树高 $H-1$ 次）。
* **性质 4（与堆的区别）**：
  败者树是自底向上调整，胜者上浮，每上一层只需和父结点（上一轮的败者）比 **1 次**；而堆是自顶向下调整，每一层需要与左右两个孩子比较，共需比 **2 次**。

## 4. 最佳归并树（优化阶段二：使归并 I/O 次数达到最少）

#### 🔑 核心目的
当利用置换-选择排序得到的初始归并段长度（权值）不等时，为了使总的 I/O 次数最少，应模仿**哈夫曼树**的思想构建最佳归并树：**让长度小的归并段先归并（处于更深层），长度大的归并段后归并（处于更浅层）**。

#### ⚠️ 严密 $k$ 叉树的虚段补充
最佳归并树必须是一棵**严格的 $k$ 叉树**（即每个非叶子结点的度数都必须正好为 $k$，不允许出现只有 2 路参与归并的秃枝）。若初始段个数 $n_0$ 不满足条件，必须补充长度为 $0$ 的“虚段”。

#### 📐 虚段判定公式
设初始归并段个数为 $n_0$，归并路数为 $k$。计算模值 $u$：
$$u = (n_0 - 1) \pmod{k - 1}$$

* 若 $u = 0$：无需添加虚段，初始段正好可以构建一棵严格的 $k$ 叉树。
* 若 $u \neq 0$：必须添加 $m$ 个长度为 0 的虚段。虚段个数公式为：
  $$m = (k - 1) - u$$

## 5. 核心公式

| 计算目标 | 核心数学公式 | 符号释义 |
| :--- | :--- | :--- |
| **初始归并段个数 $r$** | $r = \lceil \frac{N}{L} \rceil$ | $N$: 记录总数；$L$: 初始归并段大小 |
| **多路归并趟数 $S$** | $S = \lceil \log_k r \rceil$ | $r$: 初始归并段数；$k$: 归并路数 |
| **总磁盘 I/O 次数** | $\text{I/O} = 2P + 2P \times S = 2P(S + 1)$ | $P$: 磁盘物理块数；$S$: 归并趟数 |
| **败者树完全二叉树高度 $H$** | $H = \lceil \log_2 k \rceil + 1$ | $k$: 归并路数（叶子结点数） |
| **最佳归并树虚段个数 $m$** | $m = (k-1) - [(n_0 - 1) \pmod{k-1}]$ | $n_0$: 初始段个数；$k$: 归并路数 |
| **最佳归并树非叶结点数 $n_k$** | $n_k = \frac{n_0 + m - 1}{k - 1}$ | $n_k$ 即多路归并需要合并的总次数 |

## 6. 408 外部排序解题通法

### 题型一：求解总磁盘 I/O 次数
1. **求磁盘块数 $P$**：
   $$P = \lceil \frac{\text{总记录数 } N}{\text{磁盘物理块大小 } B} \rceil$$
2. **求初始段个数 $r$**：
   若使用常规内存排序：$r = \lceil N / \text{内存工作区大小 } M \rceil$。
3. **求归并趟数 $S$**：
   $$S = \lceil \log_k r \rceil$$
4. **无脑套入 I/O 总公式**：
   $$\text{I/O 次数} = 2P \times (S + 1)$$
   *(注：前部的 $2P$ 是生成初始段时的读写，后部的 $2P \times S$ 是 $S$ 趟归并的读写)*

### 题型二：手工构建最佳三叉归并树（WPL 计算）
1. **判断并补虚段**：
   计算 $u = (n_0 - 1) \pmod{k - 1}$，若 $u \neq 0$，先在待归并段队列中补上 $m = (k - 1) - u$ 个权值为 `0` 的初始段。
2. **构建哈夫曼树**：
   每次从队列中选出 **$k$ 个权值最小**的结点合并，合并生成的新结点再扔回队列，直至最后只剩一个根结点。
3. **计算 WPL（读取记录的总数）**：
   * **方法 1（定义法）**：$\text{WPL} = \sum (\text{叶子结点的权值 } w_i \times \text{到根结点的路径长度 } d_i)$。
   * **方法 2（分支累加法，推荐）**：将**所有非叶子结点**的权值全部相加，其和即为 WPL。
4. **换算为归并阶段的 I/O 次数**：
   $$\text{归并阶段的 I/O 次数} = 2 \times \text{WPL}$$
   *(注：若题目问的是“外部排序总 I/O”，别忘了还要加上生成初始归并段阶段的 $2P$ 次 I/O)*
