# 散列表 (Hash Table)

## 一、 散列函数的构造方法

构造散列函数的目标是：**计算简单**且**地址分布均匀**（减少冲突）。

### 1. 直接定址法
* **公式**：$H(key) = a \cdot key + b$ （$a, b$为常数）
* **特点**：单调、线性、不易产生冲突。
* **适用场景**：地址大小与关键字集合大小基本相同的情况。若关键字分布不连续，会造成极大的空间浪费。

### 2. 除留余数法 (★ 408最常用)
* **公式**：$H(key) = key \pmod p$
* **选值要求**：设散列表表长为 $m$，则 $p$ 通常取**不大于 $m$ 但最接近或等于 $m$ 的质数（素数）**。
* **目的**：选取质数可以最大程度减少冲突。

### 3. 数字分析法
* **方法**：分析关键字集合中各位数字的分布情况，选择其中**分布比较均匀**的若干位作为散列地址。
* **适用场景**：关键字集合已知，且关键字位数较多（如手机号、身份证号）。

### 4. 平方取中法
* **方法**：先求出关键字的平方值，然后取中间的几位作为散列地址。
* **适用场景**：关键字的每一位对最后结果都有影响，且关键字位数不长，但其分布未知。

## 二、 处理冲突的方法

当不同的关键字通过散列函数计算出相同的地址时，即发生**冲突**。

### 1. 开放定址法
> **通用公式**：$H_i = (H(key) + d_i) \pmod m$
> 发生冲突时，通过某种探测固定步长 $d_i$ 寻找下一个空闲单元。

* **线性探测法 (★ 重点)**
  * $d_i = 1, 2, 3, \dots, m-1$
  * **特点**：冲突时依次探测下一个相邻单元。极易产生**“聚集（堆积）”**现象，影响查找效率。
* **平方探测法 / 二次探测法 (★ 重点)**
  * $d_i = 1^2, -1^2, 2^2, -2^2, \dots, k^2, -k^2$ （其中 $k \le m/2$）
  * **特点**：可以有效避免“一次聚集”现象。
  * **缺点**：不能探测到散列表中的所有位置，但至少能探测到一半的位置。
* **双散列法**
  * $d_i = i \cdot Hash_2(key)$
  * **公式**：$H_i = (H_1(key) + i \cdot Hash_2(key)) \pmod m$
  * **特点**：偏移量由第二个散列函数计算得出，使不同同义词的探测序列不同，避免聚集。
* **伪随机序列法**
  * $d_i$ 为预先设定的伪随机数序列。

### 2. 拉链法 (链地址法)
* **方法**：将所有同义词存储在一个单链表中。散列表的每个单元只存储链表头指针。
* **优点**：
  * 无“聚集”现象。
  * 链表节点动态申请，适合无法预测表长的情况。
  * 物理删除操作简单。

## 三、 核心考点与易错点

> [!danger] 1. “聚集（堆积）”现象
> * **定义**：非同义词争夺同一个散列地址，导致本不该冲突的元素发生冲突，从而延长查找路径。
> * **产生原因**：主要由**线性探测法**解决冲突引起。因为线性探测法在冲突时只探查相邻的下一个位置，使得大批元素在表中连成一片。

> [!warning] 2. ASL (平均查找长度) 计算注意事项
> * **基础规则**：无论成功或失败，**即使在对应位置一次就定位成功（即没有发生冲突），探测次数（比较次数）也至少是 1**，不能算作 0。
> * **查找成功 ASL ($ASL_{succ}$)**：
>   * 分母：固定为**表中记录的元素个数 $n$**。
>   * 分子：每个已存入元素查找到自身所需的探测次数之和。
> * **查找失败 ASL ($ASL_{unsucc}$)**：
>   * **核心易错点**：当采用除留余数法 $H(key) = key \pmod p$ 时，查找失败的入口地址只有 $[0, p-1]$ 这 $p$ 个（即使表长 $m > p$）。
>   * 分母：**模数 $p$**，而非表长 $m$ 或记录数 $n$。
>   * 分子：计算从地址 $i$ ($0 \le i < p$) 出发，向后探测直到**遇到空单元**（或拉链法中链表为空）所需的探测次数之和。
>   * *注：拉链法的查找失败探测次数计算，通常不将“空指针”作为一次比较。*

> [!info] 3. 散列表查找效率决定因素
> 散列表的查找效率主要取决于三个因素：
> 1. **散列函数**
> 2. **冲突处理方法**
> 3. **装填因子 $\alpha$**
> 
> * **装填因子定义**：$\alpha = \frac{\text{表中记录数 } n}{\text{散列表长度 } m}$
> * **考点**：散列表的平均查找长度（ASL）**主要依赖于装填因子 $\alpha$**，而与表长 $m$ 和记录数 $n$ 的绝对大小无直接关联。$\alpha$ 越大，表中越满，发生冲突的概率越高，ASL 就越大。

> [!danger] 4. 开放定址法下的元素“逻辑删除”
> * **原则**：采用开放定址法时，**不能直接物理删除**（例如直接置空）表中已有元素。
> * **原因**：若直接将该位置置空，在查找其他通过该位置探测过去的同义词时，会因为遇到“空单元”而误判该元素不存在，从而**截断了其他同义词的查找路径**。
> * **正确做法**：在被删除元素的位置设置一个**逻辑删除标记**（如 Tombstone 标记）。
>   * **查找时**：遇到标记继续向后探测。
>   * **插入时**：可以视其为空，直接覆盖写入。
