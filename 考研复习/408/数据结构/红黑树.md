# 红黑树 (Red-Black Tree)

红黑树是一种自平衡的 **[[二叉排序树 (BST)]]**。它通过给结点着色（红色或黑色）并施加特定的约束规则，确保树的最长路径不超过最短路径的 2 倍，从而使查找、插入和删除的最坏时间复杂度稳定在 $O(\log n)$。

---

## 1. 定义及性质

### 1.1 五大核心定义
一棵合格的红黑树必须同时满足以下五条性质：
1. 每个结点或是**红色**的，或是**黑色**的。
2. **根结点**是**黑色**的。
3. **叶结点（所有的外部虚拟结点，即 NIL 结点）**是**黑色**的。
4. **不存在两个相邻的红结点**（即红结点的父结点和子结点必为黑色，红红不能相连）。
5. 对每个结点，从该结点到其所有叶结点的简单路径上，所含**黑结点的数量相同**。

> [!info] **什么是黑高 (Black-Height, bh)？**
> 指从某个结点出发（**不含该结点本身**）到达任意一个外部叶结点（NIL）的简单路径上的**黑结点总数**。

---

### 1.2 核心定理与高度性质
1. **路径长度定理**：从根结点到叶结点的**最短路径**（全为黑结点）不少于**最长路径**（红黑相间）的**一半**。
2. **高度界限**：含有 $n$ 个内部结点的红黑树，其高度 $h$ （不包括虚结点）满足：
   $$h \le 2\log_2(n+1)$$

> [!tip] **📖 核心证明思路（选择题常考）**
> * 设红黑树高度为 $h$，根据性质，其黑高 $bh$ 至少为 $h/2$。
> * 由于前 $bh$ 层可以看作是一棵全由黑结点构成的满二叉树，因此内部结点数 $n$ 必须满足：
>   $$n \ge 2^{bh} - 1 \implies n \ge 2^{h/2} - 1$$
> * 两边取对数，整理即得：$h \le 2\log_2(n+1)$。

## 2. 插入操作 (Insertion)

新插入的结点**默认设为红色**（因为红色结点不会破坏黑高的平衡，只可能引发红红冲突）。

### 插入调整决策树：
* **若插入结点（设为 $z$）是根结点** $\implies$ 直接染成 **黑色**。
* **若插入结点不为根**：
  * **分支 1：$z$ 的父结点 $z.p$ 为黑色** $\implies$ **维持红色**即可，无需调整。
  * **分支 2：$z$ 的父结点 $z.p$ 为红色**（此时必然存在黑色的祖父 $z.p.p$）：
    * **子分支 2.1：叔叔结点 $y$ 为红色**（父红叔红）
      * 🛠️ **处理**：将 **“父”和“叔”染黑**，将 **“爷”染红**。
      * 🔄 **后续**：以“爷”为当前结点 $z$，向上递归检查。
    * **子分支 2.2：叔叔结点 $y$ 为黑色**（父红叔黑）
      * **a) 父为爷的左孩子**：
        * 若 $z$ 为父的**右孩子**（LR型）$\implies$ 对 **“父”逆旋**，将其转化为 LL型。
        * 若 $z$ 为父的**左孩子**（LL型）$\implies$ 对 **“爷”顺旋**，并**调换原爷与原父的颜色**（即原父染黑，原爷染红），平衡结束。
      * **b) 父为爷的右孩子**：
        * 若 $z$ 为父的**左孩子**（RL型）$\implies$ 对 **“父”顺旋**，将其转化为 RR型。
        * 若 $z$ 为父的**右孩子**（RR型）$\implies$ 对 **“爷”逆旋**，并**调换原爷与原父的颜色**（即原父染黑，原爷染红），平衡结束。

## 3. 删除操作 (Deletion)

将待删除的结点按子结点（此处均指**非 NULL 的内部结点**）个数分为以下三类进行处理：

### 3.1 待删结点的分类处理
1. **只有一个子结点时**：
   * 待删除结点只能是**黑色**，其唯一的子结点必为**红色**（否则无法满足黑高平衡）。
   * 🛠️ **处理**：用其红色子结点替换待删结点，然后将该子结点**涂黑**即可。
2. **无子结点时**：
   * ① 若待删结点为**红色**：直接删除即可。
   * ② 若待删结点为**黑色**：直接删除会破坏路径黑高。需将其作为失衡结点 $N$，进入 **3.3 删除平衡操作**。
3. **有两个子结点时**：
   * 🛠️ **处理**：使用其**直接前驱**或**直接后继**作为替代删除结点，问题转化为删除该前驱或后继（必然退化为上述第 1 或第 2 种情况）。

---

### 3.2 删除平衡决策树（针对无子结点的黑色结点 $N$ 失衡）

* **若 $N$ 为根结点** $\implies$ 直接删除，无需调整。
* **若 $N$ 不为根结点**：
  * **分支 1：兄弟结点 $S$ 为红色**（此时父结点 $P$ 与兄弟的孩子 $SL, SR$ 必为黑色）：
    * 若 $S$ 为 $P$ 的左孩子 $\implies$ 对 $P$ 执行**顺旋**。
    * 若 $S$ 为 $P$ 的右孩子 $\implies$ 对 $P$ 执行**逆旋**。
    * 🔄 **后续**：**交换 $P$ 和 $S$ 的颜色**，问题转化为“兄弟 $S$ 为黑色”的情形。
  * **分支 2：兄弟结点 $S$ 为黑色**：
    * **子分支 2.1：$S$ 的两个孩子 $SL, SR$ 均为黑色**：
      * 若父结点 $P$ 为**黑色** $\implies$ 将 **$S$ 涂红**，然后以 **$P$ 为新的当前结点 $N$**，向上递归。
      * 若父结点 $P$ 为**红色** $\implies$ 将 **$S$ 涂红，$P$ 涂黑**（即交换它们的颜色），平衡结束。
    * **子分支 2.2：$S$ 的两个孩子 $SL, SR$ 不全为黑色**（即存在红孩子）：
      * **a) 同侧红孩子（或双红）**（指 $S$ 是左子且 $SL$ 为红；或 $S$ 是右子且 $SR$ 为红）：
        * 若 $S$ 为左孩子 $\implies$ 对 $P$ 执行**顺旋**。
        * 若 $S$ 为右孩子 $\implies$ 对 $P$ 执行**逆旋**。
        * 🛠️ **调整**：**交换 $P$ 和 $S$ 的颜色**，并将红孩子染成**黑色**，平衡结束。
      * **b) 异侧红孩子**（指 $S$ 是左子且 $SR$ 为红；或 $S$ 是右子且 $SL$ 为红）：
        * 若 $S$ 为左孩子 $\implies$ 对 $S$ 执行**逆旋**。
        * 若 $S$ 为右孩子 $\implies$ 对 $S$ 执行**顺旋**。
        * 🛠️ **调整**：**交换 $S$ 与红孩子（即 $SR$ 或 $SL$）的颜色**，问题转化为情形 **a)**。
